🛠️ คู่มือการติดตั้งและแก้ปัญหาการดาวน์โหลดโมเดล Ollama

หากคุณพบปัญหา max retries exceeded หรือ Connection failed ขณะรันคำสั่ง ollama pull ในออฟฟิศ แสดงว่าเครือข่ายของบริษัทอาจมีการบล็อกการดาวน์โหลดไฟล์ขนาดใหญ่จาก Cloudflare/AWS

คำแนะนำก่อนเริ่ม: ตรวจสอบให้แน่ใจว่าคุณได้ปิดและเปิดโปรแกรม Ollama ใหม่หลังจากแก้ไขการตั้งค่าใดๆ
วิธีที่ 1: ลองดาวน์โหลดโมเดลขนาดเล็ก (เพื่อทดสอบการเชื่อมต่อ)

บางครั้งปัญหาเกิดจากขนาดไฟล์ที่ใหญ่เกินไป ลองใช้โมเดลขนาดเล็ก (1.5B) เพื่อทดสอบว่าเครือข่ายอนุญาตให้ดาวน์โหลดหรือไม่:

ollama pull qwen2.5-coder:1.5b

หากวิธีนี้ผ่าน แสดงว่าเครือข่ายของคุณอนุญาตการเชื่อมต่อ แต่ไฟล์ 7B อาจใหญ่เกินไปจนหลุด (Timeout)

วิธีที่ 2: การดาวน์โหลดโมเดลด้วยตัวเอง (วิธีที่แนะนำที่สุดสำหรับออฟฟิศ)

หาก ollama pull ไม่ทำงาน ให้ใช้ Browser (Chrome/Edge) ดาวน์โหลดไฟล์โดยตรงจาก Hugging Face:

  1. ดาวน์โหลดไฟล์โมเดล GGUF (แนะนำ Q4_K_M):
    ดาวน์โหลด Qwen2.5-Coder-7B (4.7 GB)
  2. บันทึกไฟล์ไว้ในโฟลเดอร์โปรเจกต์ D:\dev\ai_scan_pdf
  3. สร้างไฟล์ข้อความใหม่ชื่อ Modelfile (ไม่มีนามสกุล) ในโฟลเดอร์เดียวกัน และพิมพ์ข้อความนี้ลงไป: 
    FROM "./qwen2.5-coder-7b-instruct-q4_k_m.gguf"
  4. รันคำสั่งเพื่อนำเข้าโมเดลเข้าสู่ Ollama: 
    ollama create qwen2.5-coder:7b -f Modelfile

ดาวน์โหลด Embedding Model เพิ่มเติมด้วยวิธีเดียวกัน:

ดาวน์โหลด Nomic-Embed (150 MB)
วิธีที่ 3: ตั้งค่า Proxy (สำหรับเครือข่ายที่มี Proxy Server)

หากบริษัทของคุณบังคับให้ใช้ Proxy ในการออกอินเทอร์เน็ต ให้ตั้งค่าดังนี้:

  1. ไปที่ Search ใน Windows พิมพ์คำว่า Environment Variables
  2. เลือก Edit the system environment variables
  3. คลิกปุ่ม Environment Variables ด้านล่าง
  4. ในช่อง System variables คลิก New:
    • Variable name: HTTPS_PROXY
    • Variable value: http://ชื่อพร็อกซี่:พอร์ต (เช่น http://10.1.1.1:8080)
  5. คลิก OK และ ปิด-เปิดโปรแกรม Ollama ใน System Tray ใหม่

สรุปคำสั่งที่ต้องใช้เมื่อติดตั้งเสร็จ

# ตรวจสอบโมเดลที่ติดตั้งแล้ว
ollama list

# รันโปรแกรม AI Assistant
python main.py

หากต้องการความช่วยเหลือเพิ่มเติม สามารถสอบถามได้ทันที!
