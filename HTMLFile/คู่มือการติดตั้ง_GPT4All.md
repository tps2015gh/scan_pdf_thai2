## ดาวน์โหลดและติดตั้ง

### ขั้นตอนที่ 1: ดาวน์โหลด GPT4All

1. เปิดเว็บไซต์: https://gpt4all.io/index.html
2. คลิกปุ่ม **"Download"**
3. เลือกเวอร์ชันสำหรับ **Windows**
4. บันทึกไฟล์ติดตั้ง

### ขั้นตอนที่ 2: ติดตั้งโปรแกรม

1. เปิดไฟล์ที่ดาวน์โหลดมา
2. คลิก **Next** ผ่านขั้นตอนการติดตั้ง
3. เลือกตำแหน่งติดตั้ง (หรือใช้ค่าเริ่มต้น)
4. คลิก **Install**
5. รอจนเสร็จ แล้วคลิก **Finish**

### ขั้นตอนที่ 3: เปิดโปรแกรมครั้งแรก

1. เปิด GPT4All จาก Start Menu
2. โปรแกรมจะโหลดโมเดลเริ่มต้น (อาจใช้เวลาสักครู่)
3. ยอมรับข้อตกลงการใช้งาน

## ดาวน์โหลดโมเดล

### โมเดลที่แนะนำสำหรับภาษาไทย

| ชื่อโมเดล | ขนาด | ความเร็ว | ความสามารถภาษาไทย |

### วิธีดาวน์โหลดโมเดล

7. รอจนดาวน์โหลดเสร็จ (อาจใช้เวลา 10-30 นาที ขึ้นกับความเร็วเน็ต)

## เปิด API Server

### ทำไมต้องเปิด API Server?

โปรแกรม AI Scan PDF ใช้ **API Server** ของ GPT4All ในการ:
- ส่งข้อความให้โมเดลประมวลผล
- สร้าง Embedding (Vector)
- ทำงานโดยไม่ต้องโหลดโมเดลใน Python

### วิธีเปิด API Server

1. เปิด GPT4All
2. คลิก **Settings** (รูปฟันเฟือง)
3. เลื่อนหาหัวข้อ **"Local API Server"**
4. เปิดสวิตช์ **"Enable Local API Server"**
5. ตรวจสอบ Port เป็น **4891** (ค่าเริ่มต้น)
6. คลิก **Save**

### ทดสอบ API Server

เปิด Browser แล้วไปที่: `http://localhost:4891`

ถ้าเห็นหน้าเว็บแสดงว่า API Server ทำงานแล้ว ✅

หรือใช้คำสั่งใน PowerShell:
```powershell
curl http://localhost:4891
```

## ทดสอบการทำงาน

### ขั้นตอนที่ 1: ตรวจสอบ GPT4All

- [ ] GPT4All เปิดอยู่
- [ ] API Server เปิด (พอร์ต 4891)
- [ ] โมเดลโหลดอยู่ (DeepSeek-R1-14B หรือ Qwen2.5-7B)

### ขั้นตอนที่ 2: ทดสอบ API

เปิด PowerShell และรัน:
```powershell
# ทดสอบส่งข้อความ
curl -X POST http://localhost:4891/api/generate `
  -H "Content-Type: application/json" `
  -d '{
    "model": "deepseek-r1-distill-qwen-14b",
    "prompt": "สวัสดีครับ",
    "stream": false
  }'
```

### ขั้นตอนที่ 3: รัน Pipeline

```powershell
# 1. แปลง PDF เป็น Chunks
python parse_and_chunk.py

# 2. สร้าง Embeddings
python embed_local.py

# 3. ถามคำถาม
python main.py
```

## แก้ปัญหา

### ❌ ไม่สามารถเชื่อมต่อ API Server

**อาการ:** `Error connecting to GPT4All: Connection refused`

**วิธีแก้:**
1. ตรวจสอบว่า GPT4All เปิดอยู่
2. ตรวจสอบว่า API Server เปิดใน Settings
3. ตรวจสอบ Port เป็น 4891
4. ลองปิด-เปิด GPT4All ใหม่
5. ตรวจสอบ Firewall ไม่ให้บล็อกพอร์ต 4891

### ❌ โมเดลไม่พบ

**อาการ:** `Model not found` หรือ `404 Model not loaded`

**วิธีแก้:**
1. เปิด GPT4All
2. ไปที่ Settings → Models
3. ตรวจสอบว่าโมเดลดาวน์โหลดเสร็จแล้ว
4. คลิกเลือกโมเดลที่ต้องการใช้
5. รอให้โหลดโมเดลเสร็จ (แถบสถานะจะแสดง "Loaded")

### ❌ ช้ามาก หรือ ค้าง

**อาการ:** ตอบช้ามาก (>30 วินาที) หรือโปรแกรมค้าง

**วิธีแก้:**
1. ปิดโปรแกรมอื่นที่ไม่ใช้ (Browser, Excel, etc.)
2. ใช้โมเดลขนาดเล็กกว่า (7B แทน 14B)
3. ตรวจสอบ RAM เหลือเพียงพอ (อย่างน้อย 4GB ว่าง)
4. Restart เครื่องเพื่อเคลียร์ RAM
5. พิจารณาอัพเกรด RAM เป็น 16GB

### ❌ หน่วยความจำไม่พอ

**อาการ:** `MemoryError` หรือ จอฟ้า

**วิธีแก้:**
1. ปิด GPT4All
2. ปิดโปรแกรมอื่นทั้งหมด
3. เปิด GPT4All ใหม่
4. โหลดเฉพาะโมเดลที่จำเป็น
5. ถ้ายังไม่พอ → พิจารณาอัพเกรด RAM

### ❌ ภาษาไทยแสดงผลผิด

**อาการ:** ตัวอักษรไทยแสดงเป็น ??? หรือ กล่อง □□□

**วิธีแก้:**
1. เปิด Command Prompt ด้วยคำสั่ง:
   ```cmd
   chcp 65001
   ```
2. ใช้ **Windows Terminal** แทน CMD ปกติ
3. ติดตั้งฟอนต์ภาษาไทย:
   - ดาวน์โหลดฟอนต์จาก Google Fonts
   - ติดตั้งใน Windows Font Settings
4. ใน Python script ตรวจสอบ `encoding='utf-8'`
