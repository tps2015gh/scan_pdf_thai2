คู่มือการใช้งานระบบ Local-Inference PDF Spec Analyzer

ระบบนี้ถูกออกแบบมาเพื่อวิเคราะห์เอกสาร PDF ขนาดใหญ่ (เช่น SRS หรือเอกสารความต้องการผู้ใช้) โดยประมวลผลภายในเครื่อง (Local) ทั้งหมด เพื่อความเป็นส่วนตัวและความปลอดภัยของข้อมูล

1. โครงสร้างของระบบ (Architecture)

  • Data Ingestion: ใช้ parse_and_chunk.py เพื่ออ่าน PDF และแบ่งข้อมูลเป็นส่วนเล็กๆ (Chunks)
  • Vector Storage: ใช้ embed_local.py เพื่อแปลงข้อความเป็น Vector ผ่าน Ollama
  • Inference Loop: ใช้ main.py สำหรับการค้นหาข้อมูลที่เกี่ยวข้องและตอบคำถามด้วย AI

2. ข้อกำหนดของระบบ (Requirements)

สำคัญมาก: จากการตรวจสอบ พบว่าเครื่องของคุณใช้ Python 32-bit ซึ่งมีข้อจำกัดด้านหน่วยความจำ (RAM 2GB-4GB) แนะนำให้ติดตั้ง Python 64-bit (เวอร์ชัน 3.10 ขึ้นไป) เพื่อประสิทธิภาพสูงสุด
  • Hardware: Optiplex 3070 (Windows 11)
  • Software: Ollama สำหรับรันโมเดลภาษา
  • AI Models:
    • qwen2.5-coder:7b (สำหรับการวิเคราะห์ Code และ Spec)
    • nomic-embed-text (สำหรับการสร้าง Vector ค้นหาข้อมูล)

3. ขั้นตอนการติดตั้งและเตรียมข้อมูล

ขั้นตอนที่ 1: ติดตั้งโมเดลใน Ollama

ollama pull qwen2.5-coder:7b
ollama pull nomic-embed-text

ขั้นตอนที่ 2: รันสคริปต์เตรียมข้อมูล

  1. แยกข้อความจาก PDF: 
    python parse_and_chunk.py
    สคริปต์จะสร้างโฟลเดอร์ chunks/ เก็บไฟล์ Markdown แยกตามหน้า
  2. สร้างฐานข้อมูล Vector: 
    python embed_local.py
    สคริปต์จะส่งข้อมูลไปให้ Ollama เพื่อสร้างไฟล์ vector_db.json

4. การใช้งานระบบสอบถาม (RAG)

รันไฟล์หลักเพื่อเริ่มการสนทนากับ AI:

python main.py
ตัวอย่างคำถาม: "ระบบมี Requirement ในการเชื่อมต่อฐานข้อมูลอย่างไร?" หรือ "ช่วยสรุปฟังก์ชันการทำงานของโมดูล A ในเอกสาร SRS นี้"

5. เทคนิคพิเศษสำหรับการใช้งานบน Optiplex 3070

  • Context Optimization: ระบบจะเลือกส่งเฉพาะส่วนที่เกี่ยวข้อง (Top 3 Chunks) ไปให้ AI เพื่อไม่ให้เครื่องทำงานหนักเกินไป
  • Memory Safety: สคริปต์มีการทำ Garbage Collection ทุกครั้งหลังประมวลผลหน้า PDF เพื่อลดการค้างของแรม
  • WSL2 Support: หากต้องการความเร็วที่มากขึ้น แนะนำให้รัน Ollama บน WSL2 (Ubuntu) ใน Windows 11
